## Trigger scenario
The Video Precheck Result webhook is triggered when the Precheck Video Content API is called, and the results are surfaced to know if the video has any violations, or if it has passed inspection.
## Data business parameters
| **Parameter name** | **Data Type** | **Sample** | **Description** |
| --- | --- | --- | --- |
| type | int | `55` | The ID of the webhook type. |
| tts_notification_id | string | `"7327112393057371910"` | The ID of the notification. |
| creator_open_id | string | `"VIyStQAAAADCBQ40s4TzOSSOEIW-bM5O9cod3vK8OytW8m-bnBYlXw"` | The open_id of the creator. To get the value, see [Authorization overview](authorization-overview-202407). |
| timestamp | int | `1644412885` | The UNIX timestamp when this webhook was triggered. |
| data | object |  |  |
| └precheck_task_id | string | `"7493990579714164574"` | ID of precheck task generated by TikTok Shop. |
| └result | string | `"FAIL"` | Possible enumerations:  <br> `PASS` <br> `FAIL` |
| └issues | []object |  | Only if the `result` is `FAIL` |
| └└risk | string | `"Pirated Content"` | Risk type. |
| └└suggestion | string | `"Your video may include unoriginal content. Creating original content is essential for standing out from the crowd."` | Suggestions on how to fix the risk. |
